# 邏輯臨床問題解決法 (Logical Clinical Problem Solving, LCPS)

本指南摘錄自《Clinical Reasoning in Veterinary Practice: Problem Solved!》，旨在提供一套結構化的臨床推論框架。

## 1. 建立並排列問題清單 (The Problem List)
*   **目標**：將模糊的主訴（背景雜訊）轉化為具體的臨床發現（硬資料）。
*   **排列優先順序**：
    *   **診斷鉤 (Diagnostic Hooks)**：具有高度特異性的症狀（如：黃疸、多渴多尿、抽搐、嘔吐）。這些問題有明確的診斷路徑。
    *   **背景資訊 (Background Noise)**：非特異性症狀（如：食慾不振、虛弱、沈鬱）。這些症狀無法指向特定系統，通常在「診斷鉤」被解決後自然得到解釋。

## 2. 定義與細化系統 (Define & Refine the System)
*   **定義系統**：哪個系統「創造」了這個症狀？（例如：嘔吐必涉及 GI 系統）。
*   **細化系統**：
    *   **原發性 (Primary/Structural)**：器官本身的解剖病變（如：異物、腫瘤、發炎）。
    *   **繼發性 (Secondary/Functional)**：器官因其他代謝問題受影響（如：肝衰竭或腎衰竭導致的嘔吐）。
*   **臨床重要性**：原發性問題通常需要「影像學」或「切片」；繼發性問題則高度依賴「生化檢驗」。

## 3. 定義位置 (Define the Location)
*   **目標**：在系統內精確定位（例如：GI 系統中的嘔吐，病灶是在胃、小腸還是大腸？）。
*   **意義**：正確的定位能避免無效檢查（例如：內視鏡伸不到回腸，若定位在下消化道則應選擇其他工具）。

## 4. 定義病灶 (Define the Lesion - DAMNIT-V)
利用此口訣窮舉可能的病因類別：
*   **D**egeneration (退化)
*   **A**nomaly (畸形)
*   **M**etabolic (代謝)
*   **N**eoplasia (腫瘤) / **N**utritional (營養)
*   **I**nflammatory (發炎) / **I**nfectious (感染) / **I**diopathic (特發)
*   **T**rauma (創傷) / **T**oxin (毒素)
*   **V**ascular (血管性)

## 5. 臨床推論查核點
*   **Chronology (時序)**：症狀出現的順序是否一致？
*   **Does this make sense?**：檢驗結果是否能解釋臨床問題？（例如：輕微升高的血糖不能解釋嚴重的沈鬱與嘔吐，必有他因）。
*   **Balance of Probabilities**：優先考慮常見病，但對不尋常的模式保持警覺（預防「隧道視野」與「確認偏誤」）。
